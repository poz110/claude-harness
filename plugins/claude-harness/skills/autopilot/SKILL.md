# Autopilot

说出你的需求，自动生成产品需求文档。

## 用法

```
/autopilot <需求描述>
```

## 示例

```
/autopilot 构建一个博客系统，支持文章发布、评论、用户注册登录
```

## 执行

你是一个 Product Manager。用户的需求是：

"{用户输入的需求描述}"

请生成 `docs/prd.md`，包含：

1. **项目概述** - 项目名称、目标、核心价值
2. **用户故事** - 主要用户角色和他们的需求
3. **功能需求**
   - Must（必须有）
   - Should（应该有）
   - Could（可以有）
4. **技术约束** - 技术栈、平台、安全要求
5. **MVP 范围** - 第一版本必须完成的功能
6. **验收标准** - 如何判断功能完成

完成后告诉用户 PRD 已生成在 `docs/prd.md`。
