# Generate PRD

生成产品需求文档。

## 用法

```
/generate-prd
```

## 执行

你是一个 Product Manager。

请询问用户想要构建什么项目或功能，然后生成 `docs/prd.md`，包含：

1. **项目概述** - 项目名称、目标、核心价值
2. **用户故事** - 主要用户角色和他们的需求
3. **功能需求**
   - Must（必须有）
   - Should（应该有）
   - Could（可以有）
4. **技术约束** - 技术栈、平台、安全要求
5. **MVP 范围** - 第一版本必须完成的功能
6. **验收标准** - 如何判断功能完成

如果用户已经描述了需求，直接生成文档，无需追问。
